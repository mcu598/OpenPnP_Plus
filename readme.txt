2023-3-7   
本人从事电子产品开发数年，最痛苦的莫过于样板焊接，发外加工成本高，速度慢，有许多时候贴片回来后依然需要手工补焊，费时费力不节省成本。有什么解决办法？国外开源的openpnp 提供了全套的解决方案，但是，配置起来非常的复杂。国内也有很多人将openpnp 稍加改造据为己有盈利销售。  我在想我们国内就没人愿意做个类似的系统用来解决这些棘手的问题，我百度了n遍均没找到相关的资源。既然，没人愿意做这种出力无利的事情，俺就试一试做第一位吃螃蟹的人。既然有想法 ，说干就干。
      预设目标是在贴片加工过程中智能化程度高，尽量减少人工参与，不追求速度，适合研发，及小批量焊接需求的解决方案，成本也要控制合理。

      硬件目前方案采用运动主控板+其他控制板模式。运动控制板使用常用GCODE作为通讯模式，cpu采用CH32V307 RISC-V芯片，开发环境RISCIDE，其他控制板采用485 modbus 通讯模式。

      软件采用C++开发，界面尽量简洁，操作容易，安全可靠。开机 加锁 不允许使用者随便改变参数，毕竟贴片机是是个复杂的系统，配置参数众多，稍有不慎参数错误，必然导致贴片加工异常。
